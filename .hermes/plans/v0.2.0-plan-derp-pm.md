## Assessment of 4 Candidates

### 1. OpenAPI Streaming Contract Tests (P2)
- **Scope:** Verify SSE streaming through the catalog routing path. Validate SSE format compliance (`data:`, `[DONE]`, chunk structure) for both legacy keyed-DI and catalog routing.
- **Effort:** ~2h. Write `StreamingContractTests.cs` with a mock `IChatClient` that streams chunks and verify the HTTP response format.
- **Dependencies:** Catalog is already wired in config. Need `WebApplicationFactory<Program>` for HTTP-level SSE tests.
- **Risk:** Low. Pattern already exists in `CatalogEndToEndIntegrationTests.cs` and `ChatCompletionsEndpoint.cs`. Just needs HTTP-level verification.
- **Status:** NOT STARTED

### 2. Provider Catalog Dynamic Reload (P1)
- **Scope:** Watch `appsettings.json` for `ProviderCatalog` section changes. Hot-reload deployments and routes without restart.
- **Effort:** ~3h. Convert `InMemoryProviderCatalog` to use `IOptionsMonitor<ProviderCatalogOptions>`. Implement `OnChange` callback that rebuilds the internal dictionaries atomically. Add concurrency tests.
- **Dependencies:** `InMemoryProviderCatalog`, `CatalogModelRouter`, DI registration.
- **Risk:** Medium. Concurrency during reload, atomicity of deployment/route swap, health state migration across reloads.
- **Status:** NOT STARTED

### 3. Rate Limiting Per Provider (P1)
- **Scope:** Token-bucket rate limiter per deployment. Enforce `MaxRequestsPerMinute` and `MaxTokensPerMinute` on each deployment. Configurable in `ProviderDeployment`.
- **Effort:** ~3h. New `RateLimitingChatClient` (DelegatingChatClient). Token bucket implementation. Config integration. Tests.
- **Dependencies:** `ProviderDeployment` config, `IChatClient` pipeline, DI.
- **Risk:** Medium. Token counting accuracy, bucket precision, concurrent access to bucket state.
- **Status:** NOT STARTED

### 4. Observability Dashboard (P2)
- **Scope:** Custom Prometheus metrics for provider health, latency, throughput. Export via OpenTelemetry. Add a `/metrics` endpoint.
- **Effort:** ~3h. Custom `Meter` with counters/gauges/histograms. Instrument `CatalogMetricsChatClient` and `HealthProbeService`. Configure Prometheus exporter.
- **Dependencies:** `CatalogMetricsChatClient`, OpenTelemetry (already wired in ServiceDefaults), NuGet packages.
- **Risk:** Medium. Need `OpenTelemetry.Exporter.Prometheus.AspNetCore` package (may not exist in .NET 10 preview). Alternative: manual `/metrics` endpoint.
- **Status:** NOT STARTED
